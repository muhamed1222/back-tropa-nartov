# 🇷🇺 Русские названия полей в Strapi

## ⚠️ Важно

В Strapi v4 **нет встроенного способа** перевести названия полей в админке через JSON схему.

Названия полей в админке Strapi берутся из:
- **API ID полей** (например, `name`, `address`, `short_description`)
- **Display Name** контент-типа (например, "Место", "Маршрут")

## ✅ Что можно сделать

### 1. Display Name контент-типов - ✅ УЖЕ СДЕЛАНО

Все контент-типы уже имеют русские названия:
- ✅ "Место" (Place)
- ✅ "Маршрут" (Route)
- ✅ "Отзыв" (Review)
- ✅ "Категория" (Category)
- ✅ "Тег" (Tag)
- ✅ "Тип" (Type)
- ✅ "Область" (Area)
- ✅ "Изображение" (Image)

### 2. Названия полей - ограничение Strapi

**Проблема:** В Strapi v4 нельзя перевести названия полей (`name`, `address`, `short_description` и т.д.) через JSON схему.

**Причина:** Названия полей берутся из API ID, который должен быть на английском для совместимости с API.

## 💡 Решения

### Вариант 1: Использовать понятные английские названия (рекомендуется)

Текущие названия полей уже понятны:
- `name` → "Название"
- `short_description` → "Краткое описание"
- `address` → "Адрес"
- `working_hours` → "Часы работы"
- `contacts_phone` → "Телефон"
- `contacts_website` → "Веб-сайт"
- `latitude` → "Широта"
- `longitude` → "Долгота"

**Преимущество:** Работает с API без изменений.

### Вариант 2: Создать кастомный плагин для перевода (сложно)

Требует разработки кастомного плагина Strapi для перевода названий полей в админке.

**Недостаток:** Сложно, требует перезапуска при изменении переводов.

### Вариант 3: Использовать описания полей

Можно добавить описания к полям, которые будут видны в админке как подсказки:

```json
{
  "name": {
    "type": "string",
    "required": true,
    "description": "Название места"
  }
}
```

Но описания - это не названия полей.

## 📝 Текущая ситуация

✅ **Готово:**
- Все Collection Types имеют русские названия ("Место", "Маршрут" и т.д.)
- Схемы настроены правильно
- API ID полей на английском (для совместимости)

❌ **Нельзя:**
- Перевести названия полей (`name`, `address` и т.д.) через JSON схему
- Это ограничение Strapi v4

## 🎯 Рекомендация

**Оставить текущие названия полей на английском**, потому что:
1. Они понятны и стандартны (`name`, `address`, `working_hours`)
2. Совместимы с API
3. Названия контент-типов уже на русском ("Место", "Маршрут")
4. Пользователи смогут понять по контексту, что означает каждое поле

Если очень нужно русские названия полей - можно:
1. Создать инструкцию для редакторов (какое поле что означает)
2. Добавить описания к полям через Content-Type Builder в админке
3. Использовать кастомный плагин (требует разработки)
