# 📍 Как добавлять места в Strapi

## 🚀 Быстрый старт

### Шаг 1: Откройте админ-панель Strapi

```
http://localhost:1337/admin
```

Войдите в систему с вашими учетными данными администратора.

---

## 📝 Пошаговая инструкция

### Шаг 2: Перейдите в Content Manager

1. В левом меню нажмите на **"Content Manager"**
2. Выберите **"Place"** из списка Content Types

### Шаг 3: Создайте новое место

1. Нажмите кнопку **"+ Create new entry"** (или **"+ Создать новую запись"**)
2. Заполните форму:

#### Обязательные поля (Required):

- **name** (Название) - например: `"Парк Атажукинский"`
- **description** (Описание) - подробное описание места
- **address** (Адрес) - например: `"г. Нальчик, ул. Толстого, 2"`
- **latitude** (Широта) - например: `43.4925`
- **longitude** (Долгота) - например: `43.6123`
- **area_id** (ID района) - например: `1`

#### Необязательные поля:

- **type** (Тип) - например: `"парк"`, `"кафе"`, `"музей"`
- **overview** (Обзор) - краткий обзор места
- **history** (История) - историческая справка
- **hours** (Часы работы) - например: `"06:00-24:00"`
- **weekend** (Выходные) - например: `"Без выходных"`
- **entry** (Вход) - например: `"Свободный"` или `"Платный"`
- **contacts** (Контакты) - телефон, например: `"+78662225566"`
- **contacts_email** (Email) - например: `"park@nalchik.ru"`
- **rating** (Рейтинг) - число от 0 до 5, например: `4.8`
- **opening_hours** (Часы открытия) - альтернативное поле
- **type_id** (ID типа) - числовой ID типа
- **is_active** (Активно) - галочка (по умолчанию включена)

#### Связи (Relations):

- **images** - Добавьте изображения (можно добавить позже)
- **reviews** - Отзывы (добавляются автоматически)
- **categories** - Выберите категории из списка
- **tags** - Выберите теги из списка
- **area** - Выберите район из списка

### Шаг 4: Сохраните и опубликуйте

1. Нажмите кнопку **"Save"** (Сохранить) - сохранит как черновик
2. Нажмите кнопку **"Publish"** (Опубликовать) - сделает запись доступной через API

⚠️ **Важно**: Только опубликованные записи доступны через API!

## 🖼️ Добавление изображений

### Способ 1: Через Media Library

1. В форме создания места найдите поле **"images"**
2. Нажмите **"Add an entry"** или **"Добавить запись"**
3. Нажмите **"Upload"** или перетащите файлы
4. Выберите изображения из Media Library
5. Сохраните

### Способ 2: Через отдельный Content Type Image

1. Создайте запись в **Content Manager** → **Image**
2. Заполните:
   - **url** - URL изображения (обязательно)
   - **alt_text** - альтернативный текст (опционально)
   - **place_id** - ID места
   - **place** - выберите место из списка
3. Сохраните и опубликуйте

## 🔗 Связывание с другими данными

### Добавление категорий

1. В форме места найдите поле **"categories"**
2. Нажмите **"Add an entry"**
3. Выберите категории из списка (можно несколько)
4. Сохраните

### Добавление тегов

1. В форме места найдите поле **"tags"**
2. Нажмите **"Add an entry"**
3. Выберите теги из списка (можно несколько)
4. Сохраните

### Выбор района

1. В форме места найдите поле **"area"**
2. Нажмите **"Select an entry"**
3. Выберите район из списка
4. Сохраните

## 📡 Проверка через API

После публикации места, проверьте через API:

```bash
# Получить все места
curl http://localhost:1337/api/places

# Получить конкретное место
curl http://localhost:1337/api/places/1

# С фильтрацией
curl "http://localhost:1337/api/places?filters[area_id][\$eq]=1"
```

---

## ⚙️ Настройка прав доступа (если API не работает)

Если API возвращает 404 или ошибку доступа:

1. **Settings** → **Users & Permissions plugin** → **Roles** → **Public**
2. Найдите **Place** в списке
3. Включите:
   - ✅ **find** (GET /api/places)
   - ✅ **findOne** (GET /api/places/:id)
4. Сохраните

Для создания/редактирования через API:
1. **Settings** → **Users & Permissions plugin** → **Roles** → **Authenticated**
2. Включите:
   - ✅ **create** (POST /api/places)
   - ✅ **update** (PUT /api/places/:id)
   - ✅ **delete** (DELETE /api/places/:id)

## 🎯 Полезные советы

1. **Всегда публикуйте записи** - только опубликованные записи доступны через API
2. **Используйте Rich Text** для описаний - поддерживает форматирование
3. **Проверяйте координаты** - используйте правильный формат (decimal)
4. **Связывайте данные** - используйте категории, теги и районы для лучшей организации
5. **Добавляйте изображения** - визуальный контент улучшает восприятие

## 📝 Массовое добавление (через API)

Если нужно добавить много мест сразу, можно использовать API:

```bash
curl -X POST http://localhost:1337/api/places \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -d '{
    "data": {
      "name": "Название места",
      "description": "Описание",
      "address": "Адрес",
      "latitude": 43.4925,
      "longitude": 43.6123,
      "area_id": 1,
      "publishedAt": "2025-11-19T00:00:00.000Z"
    }
  }'
```

## 🆘 Решение проблем

### Место не отображается в API

- ✅ Проверьте, что запись **опубликована** (не только сохранена)
- ✅ Проверьте права доступа для Public роли
- ✅ Проверьте, что `is_active = true`

### Ошибка при сохранении

- ✅ Проверьте обязательные поля (name, description, address, latitude, longitude, area_id)
- ✅ Проверьте формат координат (decimal числа)
- ✅ Проверьте, что связанные записи (area, categories) существуют

### Изображения не загружаются

- ✅ Проверьте формат файлов (JPG, PNG, WebP)
- ✅ Проверьте размер файлов (рекомендуется до 5MB)
- ✅ Используйте Media Library для загрузки
