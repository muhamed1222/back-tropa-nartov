# 🗺️ Коллекция "Маршрут"

## Структура полей

### 📝 Основная информация
- **name** (string, required) - Название маршрута (максимум 200 символов)
- **slug** (uid, required) - URL-идентификатор (генерируется автоматически из названия)
- **description** (text) - Описание маршрута (максимум 500 символов)

### 🔗 Связи с другими коллекциями
- **route_type** (relation: Many-to-One) - Тип маршрута (из коллекции "Тип маршрута": Пеший/Авто)
- **places** (relation: Many-to-Many) - Места в маршруте (из коллекции "Место")

### ⚙️ Системные поля
- **is_active** (boolean) - Активен ли маршрут (по умолчанию: true)
- **publishedAt** (datetime) - Дата публикации (draft/publish система)

---

## Как использовать в админ-панели

### 1. Создание нового маршрута

1. Откройте http://localhost:1337/admin
2. В боковом меню выберите **"Маршрут"**
3. Нажмите **"Create new entry"**
4. Заполните поля:
   - **Заголовок** (name) - обязательно
   - **Описание** (description) - краткое описание маршрута
   - **Тип маршрута** (route_type) - выберите "Пеший" или "Авто"
   - **Места** (places) - выберите несколько мест из списка
5. Нажмите **"Save"**
6. Нажмите **"Publish"** для публикации

### 2. Связи между коллекциями

#### Тип маршрута (Many-to-One)
- Каждый маршрут имеет **один** тип
- Выберите из выпадающего списка:
  - 🚶 Пеший
  - 🚗 Авто

#### Места (Many-to-Many)
- Маршрут может включать **несколько** мест
- Выберите места, которые входят в маршрут
- Места отображаются в порядке добавления

## API Endpoints

### Получить все маршруты
```
GET /api/routes?populate=*
```

### Получить маршрут по ID с местами
```
GET /api/routes/:id?populate[places][populate]=*&populate[route_type]=*
```

### Фильтрация по типу маршрута
```
GET /api/routes?filters[route_type][id][$eq]=1&populate=*
```

### Поиск маршрутов по месту
```
GET /api/routes?filters[places][id][$eq]=5&populate=*
```
