# Быстрое руководство: Работа с маршрутами

## 🚀 Создание маршрута

### 1. Подготовка мест (Places)

Перед созданием маршрута убедитесь, что у вас есть места с координатами:

```
Место 1: "Национальный музей КБР"
- latitude: 43.4981
- longitude: 43.5978

Место 2: "Эльбрус"
- latitude: 43.3499
- longitude: 42.4453
```

### 2. Создание маршрута через админ-панель

1. Перейдите в **Content Manager → Routes**
2. Нажмите **+ Create new entry**
3. Заполните основные поля:
   - **Name** - название маршрута
   - **Description** - полное описание (rich text)
   - **Type** - выберите тип маршрута (обязательно)
   - **Area** - выберите район (обязательно)
   - **Image** - загрузите обложку маршрута

4. Добавьте остановки в **Stops**:
   - Нажмите **+ Add component**
   - Выберите **Place** из списка
   - Укажите **Order** (0, 1, 2...) или оставьте пустым (установится автоматически)
   - Добавьте минимум 2 остановки

5. Нажмите **Save** - поля `distance_km`, `duration_hours` и `places_count` заполнятся автоматически!

## 📊 Пример создания маршрута

### Маршрут: "Горы Кавказа"

```
Name: Горы Кавказа
Description: Живописный маршрут по горным вершинам
Type: Пеший туризм
Area: Эльбрусский район

Stops:
  1. Place: Национальный музей КБР (order: 0)
  2. Place: Чегемские водопады (order: 1)
  3. Place: Эльбрус (order: 2)

Автоматически рассчитается:
- distance_km: ~120.50 км
- duration_hours: ~40.2 ч (для пешего маршрута)
- places_count: 3
```

## 🔧 API запросы

### Получить все маршруты с остановками

```bash
GET /api/routes?populate=stops.place,type,area,image
```

### Получить конкретный маршрут

```bash
GET /api/routes/:id?populate=stops.place,type,area,image
```

### Создать маршрут через API

```bash
POST /api/routes
Content-Type: application/json

{
  "data": {
    "name": "Горный поход",
    "description": "Описание маршрута",
    "type": 1,  // ID типа
    "area": 2,  // ID района
    "stops": [
      {
        "place": 5,  // ID места
        "order": 0
      },
      {
        "place": 8,  // ID места
        "order": 1
      }
    ]
  }
}
```

## ⚠️ Частые ошибки

### Ошибка: "Маршрут должен содержать минимум 2 места"

**Причина:** Добавлена только 1 остановка или меньше

**Решение:** Добавьте минимум 2 остановки в маршрут

### Ошибка: "В маршруте не должно быть дублирующихся мест"

**Причина:** Одно и то же место добавлено несколько раз

**Решение:** Убедитесь, что каждая остановка ссылается на уникальное место

### Ошибка: "У следующих мест отсутствуют координаты"

**Причина:** У выбранного места не заданы latitude или longitude

**Решение:** Откройте место и заполните координаты (latitude и longitude)

### Ошибка: "Некоторые места не найдены"

**Причина:** Указан ID несуществующего места

**Решение:** Проверьте, что все выбранные места существуют в базе данных

## 💡 Полезные советы

### Порядок остановок (order)

- Можно указывать вручную: 0, 1, 2, 3...
- Или оставить пустым - система установит автоматически
- Порядок определяет последовательность расчета расстояния

### Типы маршрутов и скорости

Для точного расчета времени используйте правильные названия типов:

- **"Пеший туризм"** или **"Ходьба"** → 3 км/ч
- **"Велосипедный маршрут"** или **"Велотур"** → 12 км/ч
- **"Автомобильный"** или **"На машине"** → 50 км/ч

### Изображения маршрута

- Поле `image` - для обложки маршрута (одно изображение)
- Если нужна галерея, используйте изображения из связанных мест

### Обновление координат мест

При изменении координат места (`latitude`/`longitude`):
1. Откройте маршруты, использующие это место
2. Нажмите **Save** без изменений
3. Система пересчитает `distance_km` и `duration_hours` автоматически

## 🧪 Тестирование

### Проверка автоматических расчетов

1. Создайте тестовый маршрут с известными координатами
2. Проверьте, что `distance_km` рассчитан корректно
3. Проверьте, что `duration_hours` соответствует типу маршрута

### Проверка валидации

```
Тест 1: Создать маршрут с 1 остановкой
Ожидаемый результат: Ошибка валидации

Тест 2: Добавить одно место дважды
Ожидаемый результат: Ошибка "дублирующиеся места"

Тест 3: Использовать место без координат
Ожидаемый результат: Ошибка "отсутствуют координаты"

Тест 4: Создать маршрут с 2+ уникальными местами с координатами
Ожидаемый результат: Успешное создание с автоматическими расчетами
```
